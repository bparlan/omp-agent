# manage-roadmap Skill: Strategic Roadmap Manager

## Role in OMP AEF

`manage-roadmap` is a Technical Product Manager that aligns `ROADMAP.md` with user goals, handles `/docs/ingest/` workflow with permission + context, and automatically generates the next actionable Milestone.

## Usage in Framework Skills

### When manage-roadmap is Used

| Skill | Purpose | Example Commands |
|-------|---------|------------------|
| `manage-development` | After milestone generated, start implementation | `M{X}.md` → manage-development |
| `session-audit` | After session, generate ingestion entries | `INGEST_ENTRIES.md` → manage-roadmap |
| `evolve-skills` | After ingestion processed, update framework | `ARCHIVED` → evolve-skills |

## Integration Points

### Roadmap Management Workflow

```bash
# Read project state
read "docs/ROADMAP.md"
read "docs/MILESTONES.md"

# Handle /docs/ingest/ workflow
# Refine roadmap priorities
# Generate next milestone
# Update MILESTONES.md
```

### Ingestion Workflow

**Process INGEST_ENTRIES.md:**
1. Read INGEST_ENTRIES.md
2. Present files to user with ask tool
3. Ask for permission to process
4. Ask for context (what skill/prompt to use)
5. Delegate to appropriate skill
6. Archive original files after processing

## Best Practices

### Before Managing Roadmap

**Use manage-roadmap when:**
- You need to update project priorities
- You need to handle `/docs/ingest/` files
- You need to generate the next milestone

**Avoid manage-roadmap when:**
- You need to implement features (use `manage-development`)
- You need to generate specifications (use `generate-spec`)
- You want to modify a milestone without explicit permission

### Roadmap Principles

**Always ask for human approval** before modifying the roadmap.

**Roadmap = The What & Why.** (High-level business/technical goals, epics)
**Milestone = The How Much.** (Specific, scoped deliverables extracted from the roadmap)

**Ask for permission to process /docs/ingest/ files.**

## Output

**Updated ROADMAP.md:**
- Priority changes reflected
- User-approved proposals added

**Generated Milestone:**
- File: `milestones/M{X}/M{X}.md`
- Format: Milestone template from `templates/milestone_template.md`
- Contents: Goal, motivation, scope, out of scope, success criteria, risks, notes

**Updated MILESTONES.md:**
- Appended `- [M{X}] - {goal} (active)`

**Ingestion Workflow:**
- Processed files in appropriate directories
- Original files archived

## Out of Scope

**Never:**
- Generate specifications (`M{X}S{Y}.md`)
- Write code or implement features
- Modify existing `M{X}.md` files without explicit permission
- Process `/docs/ingest/` files without user permission
