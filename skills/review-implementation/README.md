# review-implementation Skill: Analytical Review Agent

## Role in OMP AEF

`review-implementation` is an analytical skill that evaluates completed implementations against approved specifications and verification protocols without making any modifications.

## Usage in Framework Skills

### When review-implementation is Used

| Skill | Purpose | Example Commands |
|-------|---------|------------------|
| `evaluate-implementation` | After tests pass, perform analytical review | `M{X}S{Y}E.md` → review-implementation |
| `sync-documentation` | After review passes, sync artifacts to docs | `M{X}S{Y}R.md` → sync-documentation |

## Integration Points

### Review Process

```bash
# Generate review report
# Load artifacts: M{X}S{Y}.md, M{X}S{Y}V.md, M{X}S{Y}C.md
# Execute: review-implementation
# Output: M{X}S{Y}R.md
```

### Reality vs Plan Analysis

- **Completed**: Requirements fully implemented and verified
- **Partial**: Requirements partially implemented or untested
- **Missing**: Requirements not started

## Requirements

### Prerequisites

1. **Completed Implementation**
   - Specification (`M{X}S{Y}.md`) exists
   - Verification protocol (`M{X}S{Y}V.md`) exists
   - Completion report (`M{X}S{Y}C.md`) exists

2. **Implementation Files**
   - All code from `implement-specification` phase
   - Test scripts generated by `generate-tests`

### Setup

1. **Load Artifacts:**
   ```bash
   # Load milestone, spec, verification, completion
   read "milestones/M{X}/M{X}S{Y}.md"
   read "milestones/M{X}/M{X}S{Y}V.md"
   read "milestones/M{X}/M{X}S{Y}C.md"
   ```

2. **Review Scope:**
   - Execute git diff or file changes
   - Compare against specification
   - Verify test coverage

## Best Practices

### Before Performing Review

**Use review-implementation when:**
- Tests have passed (all tests passing)
- Completion report is available
- You need an unbiased analytical assessment
- Documentation needs to be updated

**Avoid review-implementation when:**
- Tests are failing or incomplete
- You need to modify implementation
- It's not time for review (evaluate hasn't run)

### Review Methodology

**Follow this strict process:**
1. Load all required artifacts
2. Execute git diff or file changes
3. Create compliance matrix
4. Compare actual tests to verification protocol
5. Document all findings
6. Write review report using template

## Output

**Review Report:**
- File: `milestones/M{X}/M{X}S{Y}R.md`
- Format: Review template from `templates/review_template.md`
- Contents: Executive summary, compliance matrix, issues found, recommendations

## Out of Scope

**Never:**
- Run the tests or evaluate results
- Modify implementation code based on findings
- Generate specifications, milestones, or verifications
- Create README.md, SUMMARY.md, .txt files, or generic documentation

## Template Reference

- **Review Template**: `~/devcode/aef/agent/templates/review_template.md`
- **Templates Directory**: `templates/` in OMP agent directory
