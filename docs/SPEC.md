# OMP System Specification

## 1. Core Principles

The OhMyPi (OMP) Framework is designed around rigorous software engineering discipline to achieve reliable and deterministic agentic workflows. Key principles include:

-   **Spec-Driven Development**: A phased, artifact-based development process.
-   **Agent-Tool Separation**: Clear demarcation between strategic agents and execution tools.
-   **Artifact Persistence**: All knowledge and decisions are recorded in immutable Markdown documents.
-   **Hierarchical Control**: A layered architecture for intent, execution, and learning.

## 2. Architectural Layers

### 2.1. Project Manager Layer

-   **Purpose**: Structures high-level intent into actionable milestones and projects.
-   **Components**: `bootstrap-project` (repo analysis), `milestone` (interactive requirements elicitation).
-   **Artifacts**: Milestone documents (`M{X}.md`), Roadmaps (`ROADMAP.md`), Project Overviews (`README.md`, `AGENTS.md`).
-   **Interaction**: Defines the scope and goals for the Tactical Lifecycle Engine.

### 2.2. Tactical Lifecycle Engine

-   **Purpose**: Executes development tasks through a deterministic, phased workflow.
-   **Workflow Stages**: 
    1.  `milestone`: Define intent.
    2.  `generate-spec`: Design the solution.
    3.  `generate-verification`: Plan tests.
    4.  `implement-specification`: Build the code.
    5.  `review-implementation`: Audit the implementation.
-   **Components**: Skills (e.g., `lsp`, `grep`, `ast_grep`, `ast_edit`, `write`, `read`, `eval`, `bash`) that agents use for execution.
-   **Artifacts**: Specifications (`SPEC.md`), Verification Protocols, Implementation Code, Review Reports.

### 2.3. Meta-Learning & Canonical Docs Layer

-   **Purpose**: Accumulates system knowledge, evolves the framework, and maintains canonical documentation.
-   **Components**: Documentation management tools, historical artifact storage.
-   **Artifacts**: `CHANGELOG.md`, `FRAMEWORK.md`, `DATA.md`, `PLAYBOOK.md`, archived milestones.

## 3. Agent-Tool Boundary

-   **Agents (Strategy)**: Interpret context, select strategies, handle ambiguity, formulate plans. Examples: `designer`, `plan`, `task`.
-   **Tools (Execution)**: Perform deterministic actions. Examples: `read`, `write`, `bash`, `eval`, `lsp`, `ast_edit`.
-   **Enforcement**: Agents MUST NOT perform tool logic, and tools MUST NOT make strategic decisions.

## 4. Public APIs and Interfaces

The primary interfaces are the OMP skill set and the defined agent roles. Agent interactions are orchestrated via the `task` tool, with inter-agent communication facilitated by `irc` and `job` management.

## 5. Data Models

Data models are implicitly defined by the specifications generated by the `generate-spec` skill and detailed in `DATA.md` if applicable (e.g., database schemas). For agentic workflows, data is primarily represented by the serialized artifacts (Markdown files) and the state managed during execution.
